공통 과정 (기초 기술)
C/C++ C와 C++은 게임 개발의 근간이 되는 프로그래밍 언어입니다. C는 저수준 메모리 접근과 빠른 성능이 필요할 때 사용되며, C++는 객체지향 프로그래밍(OOP), 템플릿, STL(Standard Template Library) 등을 추가해 대규모 게임 엔진 개발에 적합합니다. 언리얼 엔진, CryEngine 등 AAA급 게임 엔진들이 C++로 만들어져 있으며, 현대 게임 스튜디오는 C++11, C++14, C++17 등의 최신 표준을 적극 활용합니다.
WinAPI (Windows API) WinAPI는 Windows 운영체제의 기본 기능에 접근하는 API로, 게임 윈도우 생성, 기본 2D 그래픽 렌더링(GDI - Graphics Device Interface)을 담당합니다. 게임 개발에서 WinAPI는 초기 단계의 학습 목적으로 사용되는데, 간단한 2D 게임이나 프로토타입을 만들 때 유용합니다. 실제로는 DirectX 같은 고수준 그래픽 API로 이동하지만, WinAPI를 이해하면 운영체제와 그래픽 하드웨어의 상호작용을 파악하는 데 도움이 됩니다.
2D 그래픽스 프로그래밍 2D 그래픽스는 Direct2D(Microsoft의 하드웨어 가속 2D 그래픽 API) 또는 GDI를 사용합니다. Direct2D는 GPU 가속을 활용해 고품질의 2D 렌더링을 제공하며, 직선, 도형, 비트맵, 알파 블렌딩(투명도) 등을 효율적으로 처리합니다. 게임에서는 UI, HUD(Head-Up Display), 게임 오버레이 등에 2D 그래픽이 사용되므로, 이 기술을 숙달하면 게임의 인터페이스 요소들을 구현할 수 있습니다.
Blender Blender는 무료 오픈소스 3D 모델링 소프트웨어로, 게임 자산(asset) 생성의 필수 도구입니다. 게임용 3D 모델 제작 워크플로우는: (1) 고폴리 모델(세부사항이 많은 모델) 생성 → (2) 로우폴리 모델(게임 성능 최적화) 생성 → (3) UV 언래핑(2D 텍스처를 3D 모델에 매핑) → (4) 텍스처 베이킹(고폴리의 디테일을 로우폴리로 전사) → (5) 최종 텍스처 및 재질 추가 → (6) 게임 엔진 내보내기입니다. 프로그래머 입장에서는 Blender에서 생성된 FBX, OBJ 파일을 게임 엔진에 임포트하고, 물리, 애니메이션, 충돌 처리를 구현합니다.
컴퓨터 개론 (Computer Architecture) 컴퓨터 아키텍처는 CPU, 메모리, 캐시의 구조를 이해하는 과정입니다. 특히 캐시 메모리(L1, L2, L3)의 작동 원리를 알면, 게임 성능 최적화에 매우 중요합니다. 예를 들어:
	•	L1 캐시: CPU 내부, 16KB~64KB, 가장 빠름
	•	L2 캐시: 128KB~1MB, 중간 속도
	•	L3 캐시: 2MB~64MB, 느리지만 용량이 큼
게임에서는 자주 접근하는 데이터(플레이어 위치, 물리 상태 등)를 L1 캐시에 배치하도록 코드를 최적화하면 프레임 드롭을 줄일 수 있습니다.
기초수학 (3D Math Fundamentals) 게임 개발의 모든 계산은 수학을 기반으로 합니다.
	•	벡터(Vector): 위치, 속도, 방향을 표현. 벡터 더하기/빼기, 스칼라곱, 외적 등.
	•	행렬(Matrix): 회전, 이동, 스케일링 같은 변환을 수행.
	•	삼각함수(Trigonometry): 각도 계산, 회전, 원형 운동 표현.
예: 플레이어가 30도 각도로 점프할 때 초기 속도를 (v × cos(30°), v × sin(30°))로 계산합니다.
심화 과정 (고급 기술)
Modern C++ (C++17, C++20) Modern C++은 스마트 포인터(메모리 자동 관리), std::optional(값이 있을 수도 없을 수도 있는 상황 처리), constexpr(컴파일 타임 계산), 구조화 바인딩(여러 값을 깔끔하게 반환), 람다 함수 등을 포함합니다. 게임 개발에서는 이 기능들이 메모리 누수를 줄이고, 디버깅 시간을 단축하며, 성능을 향상시킵니다.
3D 그래픽스 프로그래밍 (DirectX 11) DirectX 11은 게임 개발의 핵심 그래픽 API입니다. 3D 그래픽 렌더링 파이프라인은:
	1.	버텍스 버퍼: 3D 모델의 모든 점(vertex)을 배열로 저장
	2.	인덱스 버퍼: 버텍스들이 삼각형을 이루는 순서 지정
	3.	버텍스 셰이더: 각 정점을 변환(회전, 이동 등)
	4.	래스터라이제이션: 삼각형을 픽셀로 변환
	5.	픽셀/프래그먼트 셰이더: 각 픽셀의 최종 색상 결정
간단히 말해, 게임 엔진이 “정사각형 모델을 화면에 그려”라고 하면, 이 파이프라인이 그것을 처리합니다.
시스템 프로그래밍 (Multithreading) 현대 CPU는 여러 코어를 가지고 있고, 게임도 이를 활용해야 합니다. 멀티스레딩의 활용 사례:
	•	렌더 스레드: GPU에 그리기 명령 전송
	•	게임 로직 스레드: AI, 물리 계산
	•	IO 스레드: 파일 로딩, 저장
주의할 점은 모든 것을 멀티스레드로 만들면 안 되고, 독립적인 작업들만 병렬화해야 한다는 것입니다.
3D 수학 (선형대수, 쿼터니언)
	•	벡터와 행렬: 앞서 설명한 기초 개념의 심화
	•	쿼터니언(Quaternion): 3D 회전을 표현하는 4D 수학 구조 예를 들어 캐릭터의 몸이 회전할 때, 오일러 각(X, Y, Z 회전)을 쓰면 짐벌 락(gimbal lock) 문제가 발생할 수 있지만, 쿼터니언을 쓰면 이를 피할 수 있습니다. 또한 두 회전을 부드럽게 보간(interpolation)할 때도 쿼터니언이 유용합니다.
