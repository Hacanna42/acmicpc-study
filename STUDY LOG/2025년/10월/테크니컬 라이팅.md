# **평생 가는 Vim으로 빠르게 코딩하기**
### **— IDE + Vim으로 개발 속도 향상시키기**

| 주요 독자 | IntelliJ, VSCode, Eclipse 등 IDE에 익숙한 개발자 |
| -------- | ---------------------------------------------- |
| 학습 목표 | Vim의 철학과 구조를 이해하고, 실무 개발 속도 향상을 위한 Vim 활용법을 익힌다 |
| 예상 독자 상태 | "Vim이 정말로 생산성 향상에 도움이 될까?"라는 의문을 가진 IDE 사용자 |

---

## **1. 서론: 왜 Vim인가**

### **1.1 개발 속도의 본질**

개발 속도는 단순히 타자 속도의 문제가 아닙니다.  
얼마나 빠르게 **생각을 코드로 옮길 수 있느냐**가 진짜 핵심입니다.

여기에는 두 가지 요소가 작용합니다.  
1. 커서가 코드의 목표 지점에 얼마나 빠르게 도달하는가  
2. 손이 멈추지 않고 수정, 삭제, 복사를 이어갈 수 있는가  

대부분의 IDE는 마우스 중심으로 동작합니다. 하지만 마우스를 움직이고 클릭하는 순간마다 코드 작성의 흐름이 끊기기 쉽습니다. 이런 작은 단절들이 쌓이면 몰입감과 속도 모두 떨어지게 됩니다.

---

### **1.2 마우스 중심 개발의 한계**

마우스는 편리하지만 **개발 속도 관점에서는 비효율적입니다.**  
마우스를 사용하기 위해 키보드에서 손을 떼는 순간, 손의 동선이 커지고 집중력도 분산됩니다. 한 파일 안에서 '스크롤 → 클릭 → 입력'이 수백 번 반복되면 손은 피로해지고, 실수도 잦아지죠.

반면 Vim은 이동, 편집, 명령까지 **모두 키보드로 처리**합니다.  
즉, 손이 키보드를 떠날 일이 거의 없습니다.

---

### **1.3 Vim이 해결하는 문제**

Vim은 "**손을 떼지 않는 코딩**"을 가능하게 합니다.  
모든 명령이 키보드 조합으로 이루어져 있으며, 이 조합은 의미 단위로 연결됩니다.

예를 들어 다음과 같습니다:
```
d + w  → delete word (단어 삭제)
y + $  → yank until end of line (줄 끝까지 복사)
```

이것은 단순한 단축키의 나열이 아닙니다.  
Vim은 **"편집을 위한 언어"** 에 가깝습니다.  
이 언어를 익히면 "내가 하고 싶은 편집을 문장처럼 표현" 할 수 있게 됩니다.  

이 글을 다 읽고 나면, IntelliJ나 VSCode에서도 Vim을 적용해 생산성을 눈에 띄게 높일 수 있을 것입니다.

---

## **2. Vim의 핵심 철학과 구조**

### **2.1 모드 개념 이해하기**

Vim은 하나의 상태에서 모든 기능을 수행하지 않습니다.  
**모드 기반 편집기(model-based editor)** 로, 대표적인 네 가지 모드가 있습니다.

| **모드** | **설명** | **예시** |
| -------- | -------- | -------- |
| Normal | 명령 입력 모드 | dd (줄 삭제), yy (복사) |
| Insert | 글자 입력 모드 | 일반 타자 입력 |
| Visual | 영역 선택 모드 | v 후 w (단어 선택) |
| Command | 명령 실행 모드 | :wq (저장 후 종료) |

이 구조를 이해하면 "지금 내가 어떤 상태에 있는가"를 인식하며 훨씬 체계적으로 편집할 수 있습니다.

---

### **2.2 '동사 + 명사'로 이루어진 편집 언어**

Vim의 철학은 **"문법적인 편집(language-like editing)"** 입니다.  
모든 명령은 **동사(Verb) + 명사(Object)** 형태로 조합됩니다.

| **명령** | **의미** |
| -------- | -------- |
| d + w | delete word |
| y + $ | yank to end of line |
| c + i + ( | change inside parentheses |

즉, Vim은 "명령어를 외우는 것"이 아니라 "조합하여 의미를 만드는 것"입니다.  
이 방식은 IDE 단축키처럼 개별적으로 외우는 게 아니라, **언어처럼 익히는 학습 곡선**을 만들어 줍니다.

---

### **2.3 다른 에디터와의 차이**

| **구분** | **IDE** | **Vim** |
| -------- | -------- | -------- |
| 기본 철학 | 단축키 중심 (Shortcut) | 명령 조합 중심 (Compositional Command) |
| 입력 방식 | 마우스 + 키보드 | 키보드 100% |
| 학습 곡선 | 완만하지만 장기적 제약 존재 | 급격하지만 장기적 자유 확보 |

Vim의 학습곡선은 처음엔 가파르지만, 한 번 익히면 **수십 년 동안 지속되는 속도감과 유연함**을 제공합니다.

## **3. 초급 Vim 실전: 손에 익히기**

### **3.1 이동 명령의 세계**

Vim의 기본은 **이동(motion)**입니다.  
모든 편집은 이동 명령과 결합되어 작동합니다.

| **명령** | **설명** |
| -------- | -------- |
| h, j, k, l | 좌, 하, 상, 우 이동 |
| w, b, e | 단어 단위 이동 (앞 / 뒤 / 끝) |
| 0, $ | 줄의 처음 / 끝으로 이동 |
| gg, G | 파일의 처음 / 끝으로 이동 |

Vim에서는 커서 이동이 자연스럽게 손끝에 익게 됩니다.  
마우스로 스크롤할 필요가 없다는 점에서 집중도가 높아집니다.

---

### **3.2 편집의 기본**

| **명령** | **기능** | **설명** |
| -------- | -------- | -------- |
| i, a | 입력 모드 진입 | 커서 앞 / 뒤에 입력 |
| x | 삭제 | 커서 위치 문자 삭제 |
| dd | 현재 줄 삭제 | 삭제 후 Normal 유지 |
| yy, p | 복사, 붙여넣기 | yank & paste |
| u, Ctrl + r | 되돌리기 / 다시 실행 | Undo / Redo |

이동과 편집을 결합하면 이렇게 됩니다:
```
d2w → 단어 두 개 삭제  
y3j → 현재 줄부터 세 줄 복사  
```
이런 식으로 "의미 단위 편집"이 가능해집니다.

---

## **4. 중급 Vim 실전: 속도를 만드는 기술**

### **4.1 명령의 조합과 반복**

Vim의 강력함은 "조합"과 "반복"에 있습니다.  
마지막 명령을 반복하려면 단 한 키 `.` 만 누르면 됩니다.
```
dw   → 단어 삭제  
.    → 다음 단어 삭제 (반복)
```

또한 숫자를 앞에 붙이면 반복 횟수를 지정할 수 있습니다.
```
3dd  → 세 줄 삭제  
5p   → 다섯 번 붙여넣기
```

이러한 개념은 코드 편집의 효율성을 극대화시켜 줍니다.

---

### **4.2 Visual 모드와 블록 편집**

Vim의 Visual 모드는 IDE의 멀티커서보다 유연합니다.

| **명령** | **설명** |
| -------- | -------- |
| v | 문자 단위 선택 |
| V | 줄 단위 선택 |
| Ctrl + v | 직사각형 블록 선택 |

예를 들어, 여러 줄에 동시에 주석을 추가할 수 있습니다:
```
Ctrl + v  
3j  
I# → 여러 줄의 앞에 # 주석 삽입
```
이 기능은 대규모 코드 포맷 수정에도 매우 유용합니다.

---

### **4.3 검색과 치환**

Vim의 검색 기능은 강력한 정규식 기반으로 동작합니다.

| **명령** | **기능** |
| -------- | -------- |
| / | 아래 방향 검색 |
| ? | 위 방향 검색 |
| n, N | 다음 / 이전 결과로 이동 |
| :%s/old/new/g | 전체 치환 |

예를 들어 다음과 같이 입력할 수 있습니다:
```
:%s/console.log/logger.info/g
```
이 명령 하나로 프로젝트 전체에서 로그 함수를 한 번에 교체할 수 있습니다.

## **5. 고급 Vim 실전: IDE 속의 Vim**

### **5.1 IntelliJ + IdeaVim**

Vim을 사용한다고 해서 IDE를 버릴 필요는 없습니다.  
**IdeaVim 플러그인**을 설치하면 IntelliJ에서도 Vim의 철학을 그대로 적용할 수 있습니다.

설치 후, 다음 설정 파일을 추가해보세요.
`~/.ideavimrc`:
```
set clipboard=unnamed
set number
set relativenumber
set incsearch
nnoremap <Space> :noh<CR>
```
특히 `relativenumber` 옵션은 커서 중심 거리 기반 이동을 지원하여  
`5j`, `3k` 같은 명령을 더 직관적으로 사용할 수 있게 해줍니다.

---

### **5.2 모드별 동작 차이**

| **항목** | **Vim** | **IdeaVim** |
| -------- | -------- | -------- |
| 파일 탐색 | :Ex | IDE Project 탭과 통합 |
| 플러그인 | 직접 관리 | IDE 플러그인과 연동 |
| 시각 모드 | 동일 | 멀티커서 기능과 병행 가능 |

IdeaVim은 Vim의 90% 이상 기능을 구현하고 있습니다.  
즉, **IDE의 생산성과 Vim의 유연함을 동시에 누릴 수 있는 환경**입니다.

---

### **5.3 실무 적용 전략**

1. **단계적 전환:**  
   처음부터 모든 명령을 익히려 하지 말고, 이동(`w`, `b`, `0`, `$`)부터 익숙해지세요.  
2. **Insert 모드 최소화:**  
   입력 모드보다는 Normal 모드에서 명령을 조합하는 습관을 들이세요.  
3. **매크로 활용:**  
   반복되는 편집 작업은 매크로(q로 시작)를 활용해 자동화하세요.  

```
qa      → 매크로 a 녹화 시작  
dwj     → 단어 삭제 후 한 줄 아래 이동  
q       → 녹화 종료  
@a      → 매크로 실행  
@@      → 마지막 매크로 반복
```
이 습관만 들여도 편집 속도가 2배 이상 향상됩니다.

## **6. Vim으로 얻는 변화**

Vim은 단순히 손의 속도를 높이는 도구가 아닙니다.  
오히려 **생각의 단위를 작게 만들어주는 사고 훈련 도구**에 가깝습니다.

- 코드를 '줄 단위'가 아닌 '의미 단위'로 편집하게 되고,  
- 손보다 생각이 먼저 움직이는 흐름을 익히게 됩니다.  

즉, Vim을 배운다는 것은 "편집의 문법"을 배우는 일입니다.  
IDE는 여전히 필요하지만, Vim은 **개발자의 사고방식 자체를 변화**시킵니다.

3. **IDE와 병행 학습하기**

4. **공식 튜토리얼(Vim Tutor) 활용**  
   터미널에서 `vimtutor`를 입력하면 약 30분짜리 최고의 학습 코스를 경험할 수 있습니다.
